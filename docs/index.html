Semantic Code Search for OpenCode

Ask “how does authentication work?” instead of grepping for validateToken. Your AI agent finds relevant code across your entire project — instantly.

View on GitHub Quick Start

Features

🌳

Tree-sitter AST Parsing

Extracts functions, classes, and methods as clean semantic blocks. Supports TypeScript, JavaScript, Python, and PHP.

⚡

Hash Caching

SHA-256 per-file hashes. Re-indexing only processes changed files. “157 unchanged, 2 updated” is the norm.

🔍

Natural Language Search

Describe what you’re looking for in plain English. Cosine similarity returns the closest matches across your entire codebase.

📦

Zero-Dependency LanceDB

Embedded vector store — no server, no Docker, no API key. Just works. Qdrant available for team deployments.

👀

File Watcher

Chokidar keeps the index fresh as you edit. Save a file → re-indexed in 600ms. Delete a file → blocks purged automatically.

🔄

Branch-Aware

Opt-in branch polling. Switch branches → auto re-index with hash caching. No stale blocks from the wrong branch.

Install

git clone https://github.com/jbpraxxys/opencode-indexer.git ~/opencode-indexer
cd ~/opencode-indexer
npm install
npm run build

Then add to ~/.config/opencode/opencode.json:

"plugin": [
    ["~/opencode-indexer", {
        "embedder": "openai",
        "openaiApiKey": "sk-...",
        "model": "text-embedding-3-small"
    }]
]

Quick Start

STEP 1

Opt in your project: touch .codebase-index in the project root

STEP 2

Restart OpenCode in your project

STEP 3

Install the agent skill (strongly recommended):

mkdir -p ~/.config/opencode/skills/opencode-indexer
cp skills/opencode-indexer/SKILL.md ~/.config/opencode/skills/opencode-indexer/
STEP 4

Ask your agent a natural language question:

codebase_search "how does user login work"

Auto-indexing kicks in on the first search — no manual codebase_index call needed.

Three Tools

ToolWhat it does
codebase_indexIndex the project (tree-sitter parsing + hash caching)
codebase_searchSemantic search across indexed code
codebase_statusCheck index stats and project opt-in

The agent follows a Search Priority Rule: checks codebase_status first (free, no API call), then uses codebase_search for all code search. Falls back to grep/glob only as a last resort.

CLI Tool

node cli.mjs index  ~/Sites/my-project              # Full index
node cli.mjs search ~/Sites/my-project "auth flow"  # Search
node cli.mjs status ~/Sites/my-project              # Stats
node cli.mjs clear  ~/Sites/my-project              # Delete index
